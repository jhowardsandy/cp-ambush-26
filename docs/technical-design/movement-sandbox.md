# Movement Sandbox

`Assets/Scenes/MovementSandbox.unity` is the first human-testable presentation slice. `MovementSandboxController` creates a setting-neutral eight-by-six encounter, submits authored example orders through `EncounterResolver`, and renders the emitted event log/checksum.

The sandbox is not alternate game logic: it invokes `Game.Core` directly. Use the on-screen **Resolve round** and **Reset** buttons (or `Space` and `R`) to control deterministic tick-by-tick playback. Round 1 resolves the two factions' planned movement; the gold tile costs two movement ticks and the planned blue path crosses it. After that result becomes the next planning state, Round 2 resolves a blue `field-med-kit` order and a red wait order, visibly changing blue from 8/10 to 10/10 vitality from the resolver's structured `EffectApplied` event. Round 3 resolves a named direct-fire attack; Unity animates a yellow projectile only after receiving the core `AttackResolved` event and now renders an authoritative hit or `MISS` outcome from its seeded accuracy detail. It intentionally uses primitive tiles and unit capsules; full multi-unit order authoring and richer combat remain future work.

The same scene now has a **Manual order mode**. It is a deliberately narrow first authoring surface: blue is selected, the player builds a cardinal multi-tile path and may then append one self-heal or direct attack before submitting that authored blue bundle into `EncounterResolver`; red contributes an explicit placeholder wait bundle. Each selected path step extends one `Move` action from the previous endpoint, derives its duration from terrain costs, and cannot exceed the current 10-tick round budget. A follow-up action begins after the move completes; the surface provides undo for the final action or path step. Direct attacks are intentionally speculative: they can be drafted even when red is currently out of range or sight, because the core checks range and sight only at the attack's completion tick. This does not yet make movement allowance a unit-specific catalog rule: the current authoritative cap is the round budget. Tile-click selection, multiple units, richer action timelines, target pickers, previews, and the PvE planner remain the next planning-interface slices.

`CP Ambush/Create or Open Movement Sandbox` regenerates/opens the scene from Unity's editor menu if needed.
