# Deterministic PvE Planner

`PvePlanner.Plan` is the deterministic conventional enemy-planning foundation. It reads only the supplied authoritative state, map, faction ID, named combat content, and optional roster catalog, then produces an ordinary `CommandBundle` plus one explanation per active unit. It never resolves simulation outcomes.

For each unit in stable ID order, a compatible, equipped medic first treats the legal friendly target missing the most vitality (then nearest, then stable ID). Other units select the nearest *observable* active opponent by Manhattan distance, faction ID, then unit ID. They attack if the profile declared by their unit definition is currently legal and affordable; otherwise they reserve a legal, unoccupied, distance-reducing adjacent tile. When equally distance-effective advancing tiles are available, the planner selects the highest cover value, then uses the stable north/east/south/west preference. Planned destinations are reserved during planning to avoid same-destination AI commands.

If no opposing unit is observable, the unit may make a scouting move toward caller-provided authored objectives (or the neutral map center fallback) without choosing a hidden target. Riverside Crossing supplies its `central-crossing` area. `PvePlanningPolicy` is scenario-supplied public intent, not enemy knowledge: Blue auto-play retains an occupied crossing when no legal shot exists. A `ranged` unit whose observed target is inside its selected profile minimum range instead attempts a deterministic covered retreat. The planner uses the shared visibility, effect, inventory, and attack contracts, so posture concealment affects what it can know without a second rules path. It still has no visibility memory, multi-action plans, posture-selection tactics, overwatch policy, player-selected doctrines, or randomness; those are explicit later planner inputs and policies.

Evidence: `Pve_planner_is_deterministic_and_uses_the_same_attack_command_contract`, `Pve_planner_moves_toward_nearest_target_with_an_explanation`, `Pve_planner_prefers_advancing_cover_when_distance_is_equal`, `Pve_planner_medic_treats_the_most_injured_legal_ally`, `Pve_planner_uses_the_attack_profile_declared_by_a_marksman_unit`, `Pve_marksman_repositions_when_a_target_is_inside_its_minimum_range`, `Pve_hold_policy_keeps_an_objective_occupier_in_place_when_no_shot_is_legal`, and `Pve_planner_scouts_toward_an_authored_objective_without_hidden_target_knowledge`.
